**Publish** is the story the NestRS monorepo tells. Not a crate name — a
**multi-tenant publishing platform**: publications (orgs), authors (users),
posts, live comments, background jobs, and an MCP assistant for drafting.
Several binaries, one workspace, shared `crates/features/`, no chatty RPC
between deployables.

Simple learning paths (**hello**, **blog**) live in the
[docs](/getting-started/) and the [CLI](/cli/) — scaffold them in your own
workspace with `nestrs new`. This repository hosts only the **complex**
reference: Publish.

## The apps

| App | Port | Role in the story | Transports |
|-----|------|-------------------|------------|
| [`publish-auth`](/security/authentication/issuer-and-resource-server/) | 3001 | Authors sign in (OAuth2, JWT issuance) | HTTP |
| [`publish-api`](/tutorial/) | 3002 | Main API — users, orgs, audio hooks | HTTP, GraphQL, OpenAPI |
| [`publish-assistant`](/mcp/) | 3003 | AI tools (summarize, tag, weather demo) | MCP |
| [`publish-live`](/websockets/) | 3004 | Live comments and notifications | WebSockets |
| [`publish-worker`](/queue/) | — | Async jobs — newsletters, audio queue | Queue, Schedule |

```bash
just db up                    # Postgres — publish-api + publish-auth
just dev publish-auth         # issuer on :3001
just dev publish-api          # resource server on :3002
just dev publish-live         # WebSockets on :3004
just dev publish-assistant    # MCP on :3003
just dev publish-worker       # headless — needs Redis
```

`publish-api` and `publish-auth` share the same database and the `features`
crate; the worker shares Redis with the API's queue producer. Tokens are
self-contained JWTs — the API verifies with a public key, never calls the
issuer over HTTP.

## Feature map

| Feature (`crates/features/`) | Publish role |
|------------------------------|--------------|
| `posts` | Blog articles — `publish-api` (HTTP, org + author scoped) |
| `users` | Authors and editors — `publish-api` |
| `orgs` | Publications / tenants — `publish-api` |
| `oauth` | OAuth2 login on `publish-auth` |
| `audio` | Media attachments + queue/schedule hooks |
| `authn` / `authz` | JWT verification and row-level policy — `publish-api`, not the tutorial blog |
