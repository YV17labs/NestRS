You ship one feature — **`posts`** — into the [**Publish**](/publish/) story's
**`blog`** app: entity, service, `PostsController`, validation, Postgres, and
an e2e test. No JWT, no GraphQL — that complexity lives in
[`api`](/publish/) and [`auth`](/publish/). Each page leaves
you with code that compiles and a command you can run.

## What you end up with

`hello` for the quick-start binary, `apps/blog/` for the tutorial app, and a
`posts/` feature with HTTP only.

## What you should know going in

The tutorial assumes you've already followed
[Getting started](/getting-started/) — a **workspace** you scaffolded with
[`nestrs new hello`](/getting-started/), the [`nestrs` CLI](/cli/)
installed, and `nestrs run dev hello` serving `Hello World` from
`crates/features/src/hello/`. Page 1 adds `apps/blog/` with
`nestrs new blog`; everything after that happens in `blog`, not `hello`.
If you cloned **this** repository to explore Publish, scaffold hello and
blog in a separate workspace — they are not hosted here. Skim
[Fundamentals](/fundamentals/), [Configuration](/configuration/), and
[HTTP](/http/) so module, provider, and controller are familiar terms.
`docker` is available for the Postgres step.

Rust 1.95+ is required. The database and testing pages need Postgres
(`nestrs run db up`); earlier pages run without one.
