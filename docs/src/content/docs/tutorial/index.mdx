You ship one feature — **`posts`** — into the [**Publish**](/publish/) story's
**`blog`** app: entity, service, `PostsController`, validation, Postgres, and
an e2e test. No JWT, no GraphQL — that complexity lives in
[`api`](/publish/) and [`auth`](/publish/). Each page leaves
you with code that compiles and a command you can run.

## What you end up with

`hello` for the quick-start binary, `apps/blog/` for the tutorial app, and a
`posts/` feature with HTTP only.

Two `curl`s against the running binary — no token:

```bash frame="terminal"
$ curl -s -X POST http://localhost:3005/posts \
    -H 'Content-Type: application/json' \
    -d '{"title":"Hello","body":"World"}'
{"id":"018f…","title":"Hello","body":"World"}

$ curl -s http://localhost:3005/posts/018f…
{"id":"018f…","title":"Hello","body":"World"}
```

And a passing e2e suite:

```bash frame="terminal"
$ nestrs run test e2e
         PASS [   …] blog::e2e posts_round_trip
```

## What you should know going in

- A **workspace** scaffolded per [Getting started](/getting-started/) —
  the [`nestrs` CLI](/cli/) installed and `nestrs run dev hello` serving
  `Hello World` from `crates/features/src/hello/`.
- **Rust 1.96+**, and enough of [Fundamentals](/fundamentals/),
  [Configuration](/configuration/), and [HTTP](/http/) that module,
  provider, and controller are familiar terms.
- **Docker** available — the database and testing pages need Postgres
  (`nestrs run db up`); earlier pages run without one.
- Page 1 adds `apps/blog/` with `nestrs new blog`; everything after that
  happens in `blog`, not `hello`.

<Aside type="note">
If you cloned **this** repository to explore Publish, scaffold `hello` and
`blog` in a separate workspace — they are not hosted here.
</Aside>
